# Virtual Data Enclave (VDE) User Handbook

Welcome to the VDE. This handbook provides guidance for navigating the environment, working with data, and staying compliant while performing analyses.

---

## 1. Introduction / Welcome

This handbook is intended for authorized VDE users. It covers essential workflows, security practices, and best practices to help you work efficiently and safely within the environment.

> [!NOTE]
> All activity within the VDE is monitored and must follow organizational policies and federal security standards.

## 3. User Accounts and Roles

### Roles in the VDE
- Analyst: Can access assigned datasets and compute resources  
- Reviewer: Can review outputs before export  
- Admin: Can configure resources and manage user permissions  

### Account Management
- Who to contact for role changes  
- How roles affect access to datasets and compute  

> [!WARNING]
> Never attempt to circumvent role-based restrictions. Unauthorized access is strictly prohibited.

## 4. Working with Data

### Accessing Datasets
- Approved datasets only  
- Directory structure conventions (`/project/data/raw`, `/project/data/processed`)  

### Data Handling
- Keep sensitive data in assigned directories  
- Never export data without authorization

## 8. Security and Compliance

### Session Management
- Log out when finished  
- Auto-logout occurs after inactivity  

### Data Security
- Use only approved storage locations  
- Follow guidelines for temporary and persistent files  

### Incident Reporting
- Report suspicious activity immediately to your project team  

> [!NOTE]

## 10. Troubleshooting / Support

- Common issues:
  - VM performance slow  
  - Missing dataset  
  - Application crashes  

- Contact your project team for:

  - Software errors  
  - Data problems  

> [!NOTE]

## 11. Tips / Best Practices

- Organize files and directories by project and date  
- Use descriptive file names with hyphens (e.g., `2026-03-20-analysis-results.csv`)  
- Use version control or templates for scripts when possible  
- Close idle sessions to maintain security and resource availability  

> [!TIP]
> Follow the handbook naming and folder conventions to ensure consistency across projects.
