# Basic Navigation (ADRF Environment)

This section provides a quick overview of how to navigate your ADRF project workspace after you have been granted access. 

> [!NOTE]
> See [Glossary: Project Workspace](/docs/glossary.md#project-workspace) for more information on ADRF Project Worksapces.

> [!TIP]
> You can find a **video walkthrough** video walkthrough on how to navigate the the ADRF in the `ADRF_Documentation\Navigate the ADRF` folder in the `G:` drive. 

## Workspace Overview

Your ADRF project workspace functions similarly to a standard desktop environment, with the following key areas:

- **Home / User Directory** — Your personal working space for code and intermediate files  
- **Project / Shared Drives** — Shared locations for team data and collaboration  
- **Approved Tools** — Pre-installed software for analysis (e.g., statistical or programming tools)  

For more information, see
- [Accessing Your Project Workspace](../workspace/accessing-workspace.md)


## Working with Files

- Store active work in your user directory unless otherwise instructed  
- Use shared/project folders for collaboration with approved team members  
- Follow all data handling guidelines when accessing or modifying files  



## Running Analyses

- Launch approved tools from the application menu or desktop  
- Perform all analysis within the VDE environment  
- External connections and downloads are restricted unless explicitly authorized  

## Saving and Outputs

- Save work frequently within your assigned directories  
- Outputs intended for export must follow the required review and approval process  
- Do not attempt to move files outside the environment outside of approved workflows  

## Session Management

- Log out when your session is complete  
- Sessions may time out after periods of inactivity  
- Unsaved work may be lost if a session ends unexpectedly  

## Tips for New Users

- Keep files organized within your user or project folders  
- Name files clearly to support review and collaboration  
- Reach out early if you encounter access or performance issues
